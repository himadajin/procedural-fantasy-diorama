# procedural-fantasy-diorama アートディレクション

この文書は、procedural-fantasy-diorama の絵づくり(3Dレンダリングの見た目、
画面構成、UIの造形)の規範を定義する。

- 「どのようなアプリであるか」は `design.md` を正とする。
- 「どう作り、どう検証するか」は `implementation-spec.md` を正とする。
- 本書は「どう見えるべきか」を数値と規律のレベルで固定する。
  実装中に見た目の判断で迷った場合は本書に立ち返る。
  パレットや光の数値を変更したくなった場合は、コードより先に本書を更新する。

---

## 1. シグネチャ

このアプリの絵は、次の一枚に集約される。

> 夕霧の稜線 —
> 低い暖色の斜光が屋根の稜線・塔の頂・水面だけを拾って温かく光らせ、
> 建物の足元と箱庭の外縁は青灰の霧へ冷たく溶けていく。
> その霧の中に、結界の青緑がぽつぽつと冷たく灯る。

`design.md` の「夕方から曇天寄りのファンタジーゲーム風ライティング」
「軽い霧」「石と木と屋根が映える色調」「二つの光の対比」という指定を、
この一文まで先鋭化する。

この絵の核は二つの光の対比である。
太陽と人の灯りは暖かい琥珀、魔法の光は冷たい青緑。
暖色が生活を、青緑が魔法を担当し、互いの領分を侵さない。

全ての色・光・カメラ・UIの決定はこの絵に奉仕する。
大胆さは3Dシーンに集中させ、UIは徹底して寡黙に保つ。
迷ったときの判定基準は「その変更は夕霧の稜線を強めるか、薄めるか」である。

## 2. 色の規律

### 2.1 色相族の制限

全体を3つの色相族+2アクセントに制限する。

- 冷: 青灰族 — 空、霧、水の深み、スレート屋根、石、UIの補助テキスト
- 暖: 黄土〜褐色族 — 斜光、漆喰、木部、土、砂洲、瓦
- 緑: 苔色族 — 草地、樹冠。純緑ではなく灰と黄土に寄せる
- アクセント1: ランタン琥珀 — 太陽光の色。水面のハイライトとUIのアクセントを含む
  暖色側のすべてを担い、画面全体を一つの太陽で貫く。
  窓明かりなど琥珀の発光(emissive)は扱わない。シーンで発光するのは魔法(青緑)のみ
- アクセント2: 結界青緑 `#5fd4c0` 系 — 魔法の光専用。
  結界門の光、魔導塔の頂、発光する紋様、水晶、魔法灯、浮遊要素、
  魔導系中心建築の発光開口・結界紋様にのみ使う

結界青緑の使用規則:

- 発光する要素(emissive)にのみ使う。非発光の面には使わない
- 常に小面積に留める。壁や屋根など大面積の面に青緑を塗らない
- UIには使わない。UIのアクセントは琥珀一系統を守る
- 太陽の琥珀と同一画面で共存させ、暖と冷の対比として機能させる

大面積(地面、壁、屋根、水面)の彩度はHSVで35%程度を上限とする。
瓦の基準色のみ、屋根面に限る例外として超過を許す。
発光する小面積の魔法要素もこの上限の例外とする。
様式化は彩度の高さではなく、明度設計とシルエットで作る。

### 2.2 明度のヒエラルキー

暗い順に固定する。

```
水面 < 屋根 < 壁 < 地面 < 空
```

この階段は、日陰面・霧中での見え(拡散照明下の描画結果)を指す。
基準色どうしのアルベド比較ではなく、寒色の環境光の下で
上を向く地面が垂直の壁より強く照らされる状態で判定する。
太陽の受光面が階段を跳び越えて明るく光ることは例外であり、
それこそがシグネチャの稜線を作る。
浅瀬の明るい色は水際の細い帯に限り、水面の大部分は深みの暗さを保つ。

この階段により、どの俯瞰角度でも屋根と塔のシルエットが
空と地面の両方から分離する。
新しい要素(植生、結界構造、橋)を追加するときは、
必ずこの階段のどこに属するかを決めてから色を選ぶ。
樹冠は「屋根と壁の間」、結界壁と魔導塔の躯体は「壁」、
橋は等級に応じて「壁〜地面」に属する。

発光する魔法要素(結界の光、水晶、魔法灯)はこの階段の例外であり、
シーンで最も明るい点として扱う。
例外である以上、面積を絞り、数を絞る。
発光面が大きくなるほどシグネチャは薄まる。

## 3. ライティング・レシピ

PHASE 1 の時点で最終品質まで実装し、以降のPHASEでは原則として変更しない
(PHASE 6 の総合調整でのみ、本書を更新した上で微調整を許す)。

- 平行光(太陽): 色 `#ffc189`、強度 1.15、仰角 約24度。
  低い仰角が長い影と稜線のリムを作る。方位は seed に依らず固定
- HemisphereLight: 空 `#8ea6c0` / 地 `#665d4e`、強度 0.5。
  日陰を青く、地面からの照り返しをわずかに暖かく
- FogExp2: 色 `#a9b6c6`、密度は `2.2 / ワールド一辺` を基準に調整。
  霧色は地平の空色と連続させ、外縁の溶け際を作る
- トーンマッピング: ACESFilmic、露出 1.05
- 影: 平行光1灯のPCFソフトシャドウ。
  シャドウカメラはワールド境界にフィットさせる
- 接地感は影だけに頼らない。頂点カラーAO
  (壁の足元、軒下、石垣の目地、樹冠の下面を一段暗く焼き込む)で補強する
- 魔法の光は emissive(自己発光)のみで表現し、動的光源を追加しない。
  ポストプロセスのBloomも使わず、ACESトーンマッピングの範囲で光らせる。
  魔法灯や結界門の足元には、ごく弱い青緑の照り返しを
  頂点カラーで焼き込んでよい(半径は小さく、彩度は控えめに)

## 4. 空

画像テクスチャは使わないため、内向き球にシェーダーでグラデーションを描く。

- 天頂: `#45566e`
- 中空: `#8195ab`
- 太陽側の地平: `#d9a978`(暖)
- 反対側の地平: `#96a4b3`(冷)

太陽側だけ地平が暖かいことで「夕方」が一目で読め、
霧色 `#a9b6c6` との連続性も保たれる。

## 5. マテリアルパレット

頂点カラーの基準値。実際の生成では seed とパラメータによる
±8%程度の明度ムラ・個体差を各基準値の周りに与える。

### 5.1 建築

- 石(結界壁・塔・基壇): `#8e8a84`、目地・下端の焼き込み `#6f6d6a`、
  切石(高Prosperity) `#a8a49b`
- 漆喰: `#d9cfbc`、汚れムラ `#c2b7a3`
- 木部(梁・杭・柵・欄干): `#5d4a38`
- 粗い木の壁(最下層Prosperity): `#6e5a44`。木部より一段明るく、茅屋根より暗い
- 屋根: スレート `#5c6b7a` / 瓦 `#a0563c` / 木羽 `#7a6a52` / 茅 `#97855a`。
  Prosperityの階層でパレットが茅→木羽→瓦→スレートへ移行する。
  同一屋根内でも面ごとに±8%の明度ムラを入れる

### 5.2 地面と道

- 草 `#7a8a55`、土 `#8a7458`、砂洲 `#a89570`、湿地 `#6b7452`
- 道: 土 `#8a7458` → 砂利 `#9a927f` → 石畳 `#7e7c76`(Prosperity駆動)
- 広場の舗装: 石畳と共通
- 地面は平坦だが単一色にしない。水辺、道路、広場、建物密度と関係した
  材質ゾーンのブレンドで、板ではなく土地として読ませる

### 5.3 水と植生

- 水: 深み `#2e4a5c`、浅瀬 `#5f8494`(青灰寄り)。
  水面には青緑の純色を置かず、必ず灰を混ぜて結界の光と混同させない。
  太陽方向のハイライトにのみ琥珀を混ぜる
- 水路: 水色は川と共通。縁石 `#7e7c76`、護岸の石積み `#8e8a84`、杭 `#5d4a38`
- 樹冠: `#5f7a4a`〜`#45603c` の個体差、幹 `#55452f`
- 低木・草むら: `#6b7f52` 前後。樹冠と同域でやや明るく

### 5.4 魔法と結界

- 発光コア(水晶の芯、結界門の光、魔法灯の火屋): `#5fd4c0`
- 発光の縁・弱い光(紋様のライン、光の減衰部): `#8fe6d8`
- 発光開口(魔導系中心建築の窓): コア `#5fd4c0`。開口の奥に発光面を置く
- 水晶の非発光部: `#4a6a78`(青灰族に寄せた鉱物色)
- 結界石・境界標・浮石の躯体: 石 `#8e8a84` と共通。
  発光する紋様が刻まれる部分にのみ青緑を使う
- 魔法灯の柱: 木部 `#5d4a38` または石 `#8e8a84` と共通。祠・立石も石と共通
- 発光要素は明度階段の例外(2.2節)。面積と数を絞り、
  霧の中に点在する冷たい灯りとして配置する

発光の数値規範:

- 発光コアの emissiveIntensity は基準 2.0(明滅は ±0.3、周期4〜7秒)、
  縁・紋様は 0.6〜0.8
- ACESトーンマッピングによるコアの白化は許容する。青緑の同定はコアではなく、
  縁 `#8fe6d8` と足元の照り返しが担う
- 発光面のベース(拡散)色は `#2a3a38` 程度の暗色に固定し、
  夕光の加算で濁らせない
- 1要素の発光面の寸法は建物1階高の1/3以下。Arcana 最大時でも
  発光要素の合計投影面積は箱庭平面の0.5%以下とし、導出設定で上限をクリップする

## 6. 造形の様式言語

写実の縮小ではなく「絵本の断面模型」に寄せる。
テクスチャなし・頂点カラーのみで様式として成立させる鍵は、
少し大げさな比率である。

- 屋根勾配は50〜58度と急に。軒の張り出しは深めに
- 煙突は実寸比で15%ほど過大に
- 窓は現実より少なく大きく。壁は厚く、開口はセットバックさせて奥まって見せる
- 樹木は幹+2〜3個の不整形な塊の樹冠で、やや大ぶりに
- 石垣・縁石などの反復部品は、細かく刻むより一つ一つを大きめに読ませる
- 魔導塔は一般建物より明確に細長く、頂部に水晶や尖った屋根を持つ。
  高さの誇張で「魔法の塔」であることをシルエットだけで読ませる
- 浮遊要素は小さな水晶・浮石を数個のクラスタで。
  大きな一つより、小さな数個が緩く並ぶ方が魔法として読める

## 7. 水面の見せ方

半透明に頼らず、色で読ませる。

- 岸距離属性による浅瀬→深みのグラデーション
- 低周波ノイズによる緩い法線揺らぎ
- フレネル風の縁の明るみ
- 太陽方向にだけ琥珀のスペキュラ筋

水面は地面より一段低く置き(目安は壁1階高の1/5。実装値は
`implementation-spec.md` の 0.6 単位)、縁石・護岸・土手の段差で水際を読ませる。
この段差は平坦な箱庭に残る数少ない高低差であり、奥行きの手がかりとして扱う。

水面単体で完成させない。橋脚、杭、砂洲、湿った岸、岸辺植生と
組み合わせて初めて「水辺」として読める、という前提で設計する
(`design.md` 水辺の表現に対応)。
街中の水路は、縁石・護岸・岸沿いの道・小さな橋との組み合わせで
「街の一部としての水」として読ませる。

## 8. カメラの初期構図

- FOVは40度。やや望遠寄りの圧縮でミニチュア感を出す
- 初期視点は太陽方位から約120度ずれた方角、俯角45度前後から
  中心建築(生成前は箱庭中心)を見下ろす構図
- この角度で屋根の受光面と影面が同時に見え、稜線のリムが最も効く
- Reset Camera はこの構図に戻る

## 9. UIのデザイン

### 9.1 方針

UIは夜景の中の計器盤である。暗く、静かに、3Dの露出を邪魔しない。
明るいパネルは夕暮れの絵の露出を壊すため使わない。
アクセント色はシーンの太陽光と同族の琥珀に固定し、
UIを「この絵の一部」として成立させる。
結界青緑はシーン内の魔法専用であり、UIには使わない。

### 9.2 UIトークン

```
--panel-bg:      rgba(16, 20, 26, 0.88)   /* backdrop-blur 12px と併用 */
--panel-border:  rgba(255, 255, 255, 0.07)
--panel-radius:  14px
--text-hi:       #e8e4da   /* 漆喰と同族の暖白。純白は使わない */
--text-lo:       #98a0ab   /* 霧と同族の青灰 */
--accent:        #d79a4a   /* ランタン琥珀。太陽光 #ffc189 の暗所適応版 */
--accent-press:  #b87f39
--track:         rgba(255, 255, 255, 0.14)
--hairline:      rgba(255, 255, 255, 0.06)
```

### 9.3 タイポグラフィ

- 表示書体: IM Fell English(Google Fonts)。
  使用箇所はタイトル「Procedural Fantasy Diorama」と
  サマリー先頭の生成記述文の欧文のみ。ウェブフォントはこの1書体に限る。
  読み込めない環境や読み込み失敗時は Georgia 等の serif スタックへ
  フォールバックして続行する(font-display: swap)。フォールバックは許容仕様とする
- UI本文: system-ui スタック。日本語は Hiragino / Noto Sans JP 系へ
  自然にフォールバックさせる
- データ: ui-monospace スタック。seed 入力と数値
  (パラメータ値、建物数、頂点数)は等幅で組む。
  seed が「データである」ことを書体で示す

### 9.4 レイアウト

パネルは画面端にドッキングせず、余白を持って浮くカードにする。
3Dが四辺すべてで見え続け、「箱庭が主、パネルが従」が構造で伝わる。

デスクトップ(右サイドパネル、幅約320px):

```
┌────────────────────────────────────────────┐
│  ◦Generating…(生成中のみ)      ┌────────┐ │
│                                  │ 題字    │ │
│                                  │ seed ▢🎲│ │
│           3 D ビ ュ ー           │ ──────  │ │
│                                  │ 6スライダ│ │
│                                  │[Generate]│ │
│                                  │ ──────  │ │
│  ⌂(Reset Camera)                │ サマリー │ │
│                                  └────────┘ │
└────────────────────────────────────────────┘
```

モバイル(ボトムシート、半開/全開の2段階):

```
┌──────────────┐
│ ◦Generating… │
│              │
│   3Dビュー    │
│           ⌂  │
│┌────────────┐│
││ ═══(つまみ) ││ ← 半開: seed + Generate のみ
││ seed ▢  🎲 ││    全開: スライダーとサマリーまで
││ [Generate] ││
│└────────────┘│
└──────────────┘
```

半開状態は「seed(Randomize Seed を含む)と Generate だけ」に絞る。
最頻の操作(引き直して眺める)が最小のUIで回り、
スライダーは意図して開いたときだけ現れる。
ブレークポイントは約768pxを目安とする。

### 9.5 コンポーネント

- Generate: 琥珀の塗りボタン。パネル内で唯一の塗り面とし、
  主アクションであることを塗りの独占で示す
- Randomize Seed: seed 欄の隣のサイコロ型ゴーストボタン。
  押しても再生成しないことを、塗りを持たない見た目でも示す
- スライダー: 3pxの細トラック、琥珀の充填、18pxのサム。
  右端に等幅の数値。直下に1行の説明を --text-lo の12pxで置く
- Reset Camera: ビューポート左下の円形ゴーストボタン。
  パネル外に置き、「カメラはビューの持ち物」であることを配置で示す
- Generating インジケーター: 上部中央の小さなピル。
  琥珀の点滅ドット+現在ステップ名(「水系を計画中…」「結界を編成中…」)。
  パイプラインのステップ名をそのまま流用し、進捗表示を生成過程の物語にする
- サマリー: 先頭に生成記述文(例「水路の交わる魔法都市。三基の魔導塔と結界環を持つ」)を
  やや大きい暖白で置き、以下はラベル(青灰)+値(暖白、数値は等幅)の行を
  ヘアラインで区切る

### 9.6 初回起動の演出

初回ロードでは、空のグラデーションと霧、地平の暖色だけが見える
「世界が生まれる前」の画面にインジケーターのみを重ねる。
生成完了で箱庭がフェードインする。
設定画面を先に見せない(`design.md` 中心となる体験に対応)。

### 9.7 品質の床

宣言せず黙って満たす。

- キーボードフォーカスの可視化
- prefers-reduced-motion でカメラ慣性・点滅・フェード・浮遊要素の上下動・
  魔法光の明滅を減衰
- タッチターゲット44px以上
- パネルの開閉状態に関わらず3Dビューの操作が生きている

## 10. PHASE別の絵づくり検収観点

`implementation-spec.md` の各PHASE完了条件に加えて、
本書の観点から次を検収する。

- PHASE 1: 空・光・霧・トーンマッピングとUIトークン一式が本書の数値通りで、
  仮グラウンドの時点で「夕霧の空気」がすでに成立している。
  半開ボトムシートの2段階挙動が完成している
- PHASE 2: 明度ヒエラルキー(この時点では水<地面<空)が成立し、
  外縁が霧へ溶ける絵になっている。水面の琥珀ハイライトが太陽方向にのみ出る。
  平坦な地面が材質ゾーンのブレンドで土地として読める
- PHASE 3: 道と広場の色が5.2節のパレットに従い、
  地面の絵を壊さず骨格として読める。道が碁盤目に見えず有機的に曲がる
- PHASE 4a: 建物のシルエットが屋根勾配・軒・比率の様式言語に従い、
  明度ヒエラルキー(水<屋根<壁<地面<空)が完成する
- PHASE 4b: 素材パレットの階層移行(茅→木羽→瓦→スレート、
  掘立→木組み→石造→切石)が5.1節の色で現れる
- PHASE 5a: 初期構図で中心建築の稜線がリムを受け、
  シグネチャの絵が中心建築込みで成立する
- PHASE 5b: 結界壁・魔導塔・結界門・橋が明度階段の正しい位置に入り、
  結界の青緑が「霧の中に点在する冷たい灯り」として琥珀の斜光と対比する。
  発光面積が大きすぎて絵の露出を壊していない
- PHASE 6: 色相3族+2アクセントの規律を全要素に対して最終監査する。
  規律から外れた色が1つでもあれば本書を更新するか色を直す。
  結界青緑が発光要素以外に漏れていないことを確認する。
  起伏がなくても、霧・スカイライン・材質ゾーン・水面の低さによって
  奥行きが読めることを確認する
- PHASE 7: 性能プリセット(解像度・影・インスタンス数・水面シェーダー簡略度・
  発光の明滅や浮遊の更新頻度の削減)が
  絵(色・光・構図)を一切変えないことを確認する
