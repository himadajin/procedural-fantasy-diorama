## 4. 禁止事項の最終確認

- 起伏地形 — なし。地面は y=0 の平坦(zoneMask の材質差のみ)。✅
- 軍事的防衛の意味論 — なし。壁は胸壁なしの結界壁、`src/` に軍事語彙
  (防衛・籠城・城壁の military 表現)を導入する実装はない。✅
- 浮遊島・浮く建物 — なし。浮遊要素は塔・門・聖域近傍の小さな水晶のみ
  (既存テストで近傍制約を担保)。建物 footprint はすべて接地または杭支持。✅
- 外部 3D モデル・画像テクスチャ — なし。
  `TextureLoader|GLTF|FBX|OBJLoader|ImageLoader|画像拡張子` の grep が 0 件。✅
- TODO・未実装・プレースホルダー残置 — なし。
  `TODO|FIXME|HACK|XXX|placeholder|プレースホルダ|仮実装|未実装` の grep で
  実装上の残置 0 件(ヒット 2 件はいずれも「仮実装ではない」ことを説明する
  ドキュメントコメントと、段 1 実行前の初期値であることを説明するコメント)。✅
- Math.random — 使用 0 件(ヒットはすべて禁止規約を記す
  コメント)。✅
- モード切替・手動編集・人物・動物・生活小物・音 — UI/生成物に存在しない(目視+コード)。✅

## 6. 発見事項(修正は本 commit のスコープ外。要否は別途判断)

1. **サマリー数量詞の不自然さ** — 結界門が 10 以上のとき
   「10つの結界門」という表記になる(everdusk-105 で発生)。
   contracts/worldmodel.md Summary 節の「1〜9 は漢数字+助数詞、10 以上は算用数字」
   どおりの実装だが、助数詞「つ」と算用数字の組み合わせが日本語として不自然。
   契約側の規範を「10 以上は『10 の結界門』等」へ見直す余地がある。
2. **World Scale 0 の疎なセル** — seed によっては建物 2 棟まで疎になる
   (seed-b: 湖が敷地の大半を占める)。破綻ではないが「建物と街が主役」の
   下限として目視確認が望ましい。
3. **バックグラウンドタブでの生成時間** — パイプラインは段間で rAF に制御を
   返すため、タブが非アクティブで rAF が間引かれると生成が 10 秒超に伸びる
   (検証環境のプレビュータブで観測)。アクティブなタブでは影響しない設計だが、
   実機での生成時間計測(目標 5 秒以内)を確認依頼に含める。

## 7. 実機確認依頼(ユーザー実機での最終確認一式)

- デフォルト(everdusk-102・全 50)の初回表示の印象
  (中立な集落として見えるか、重すぎないか)
- タッチ操作感(1 本指回転・ピンチ・2 本指パン・Reset Camera)
- 実機 fps(操作時 30fps 以上)と生成時間(5 秒以内)、
  連続再生成でのメモリ増加なし(?debug=1 の `pfdSoak()`)
- Arcana 100 の発光・浮遊の品(露出が崩れず「霧の中の灯り」に収まるか)
- 極端セルの見え方(World Scale 0 の寒村、Water 100 の水郷)で
  「建物と街が主役」が保たれているか
